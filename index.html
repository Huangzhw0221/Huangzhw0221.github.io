1：两数之和

1. 两数之和 - 力扣（LeetCode）

题干：给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。你可以按任意顺序返回答案。

思路1：暴力法，可以用两次for循环找，由于只存在一个结果，所以从头开始找，只要过一遍就可以。这个的缺点是极慢。

思路2：哈希表，使用哈希表寻找target-x，如果存在，那就直接返回下标，如果找不到就把 x 插入哈希表，保证 x不和自己匹配。在python中使用字典就很容易实现了。

15：三数之和

15. 三数之和 - 力扣（LeetCode）

题干：给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。

思路1：排序后三重循环，假设从头开始，将右指针设定在队列末尾，将左指针设定在目标位置右边一个，将右指针逐渐左移，计算三数之和是否为0。这样可以保证循环得到的三元组不会重复，并且由于有先验知识，即在有序序列中，如果左右指针交会那么再移动也不会有结果了，就要跳出循环。所以从时间复杂度来看就是一个二重循环的复杂度。

16：最接近的三数之和

16. 最接近的三数之和 - 力扣（LeetCode）

题干：给你一个长度为 n 的整数数组 nums 和 一个目标值 target。请你从 nums 中选出三个整数，使它们的和与 target 最接近。

返回这三个数的和。假定每组输入只存在恰好一个解。

思路1：和三数之和类似，先要让数组有序。随后设定一个best作为比较对象，在双重循环时做两次判断，一次是三数之和与目标数的比较，一次是和best的比较。最后返回这个最佳值就可以了。

17：电话号码的字母组合

17. 电话号码的字母组合 - 力扣（LeetCode）

题干：给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。答案可以按 任意顺序 返回。

给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

思路1：bfs广度优先搜索。在定义了一个哈希表/字典后，用广度优先搜索，将原始字符串种的数字挨个拿出来，从第一个开始，将所有可能性填入。

18：四数之和

18. 四数之和 - 力扣（LeetCode）

题干：给你一个由 n 个整数组成的数组 nums ，和一个目标值 target 。请你找出并返回满足下述全部条件且不重复的四元组 [nums[a], nums[b], nums[c], nums[d]] （若两个四元组元素一一对应，则认为两个四元组重复）：

0 <= a, b, c, d < n
a、b、c 和 d 互不相同
nums[a] + nums[b] + nums[c] + nums[d] == target

思路1：和三数之和类似，先排序，再移动双指针。就是有点烦了。

28：找出字符串中第一个匹配项的下标

题干：给你两个字符串 haystack 和 needle ，请你在 haystack 字符串中找出 needle 字符串的第一个匹配项的下标（下标从 0 开始）。如果 needle 不是 haystack 的一部分，则返回 -1 。

思路1：逐个比对法，这个思路就是创建一个比对函数，通过递归调用这个比对函数，得到第一个匹配项的下标。在这里需要特别注意若干特殊情况，比如下标溢出就返回-1，needle比haystack还长就返回-1。

53：最大子数组之和

53. 最大子数组和 - 力扣（LeetCode）

题干：给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。子数组 是数组中的一个连续部分。

思路1：暴力求解，两个for循环语句嵌套，是一个时间复杂度度为 O(n^2) 的解法。当然这个解法是超时的。

思路2：动态规划，如果前面子数组的和大于0，那么不论正负，将下一个数字加上去；如果前面子数组的和小于0，且下一个数比这个和大，那么就不再相加，直接保留这个数。

思路3：分治法，最大子数组只可能在左边子序列、右边子序列、跨越中间的子序列。使用递归的方式，将大的序列分成两半。对于拆分成两半的，有三种可能：

1：在左边

2：在右边

3：跨越中间

那么，对于子段分别计算左最大和、右最大和即可。

题干：以数组 intervals 表示若干个区间的集合，其中单个区间为 intervals[i] = [starti, endi] 。请你合并所有重叠的区间，并返回 一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间 。

88：合并两个有序数组

88. 合并两个有序数组 - 力扣（LeetCode）

题干：给你两个按 非递减顺序 排列的整数数组 nums1 和 nums2，另有两个整数 m 和 n ，分别表示 nums1 和 nums2 中的元素数目。

请你 合并 nums2 到 nums1 中，使合并后的数组同样按 非递减顺序 排列。

注意：最终，合并后数组不应由函数返回，而是存储在数组 nums1 中。为了应对这种情况，nums1 的初始长度为 m + n，其中前 m 个元素表示应合并的元素，后 n 个元素为 0 ，应忽略。nums2 的长度为 n 。

思路1：合并后排序，使用sort方法，在两种语言中都很简单。

思路2：双指针法，利用题目中的非递减顺序，说明可以有序地获取数值，那么就设置两个指针指向两个队列的队头，比较两个队头数值的大小，把小的那个放入目标队列即可。

118：生成杨辉三角

题干：给定一个非负整数 numRows，生成「杨辉三角」的前 numRows 行。在「杨辉三角」中，每个数是它左上方和右上方的数的和。

思路1：暴力法，用循环嵌套，由于下一行首末两个数字只能获取上一行的一个数字，所以排除这两个特殊情况就可以了。

119：杨辉三角Ⅱ

题干：给定一个非负索引 rowIndex，返回「杨辉三角」的第 rowIndex 行。在「杨辉三角」中，每个数是它左上方和右上方的数的和。

思路1：暴力法，和上一题一样。

121：买卖股票的最佳时机

题干：给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

121. 买卖股票的最佳时机 - 力扣（LeetCode）

思路1：暴力法，遍历两边数组，用后项减去前项，不过复杂度是 O(n^2)，会超时。

思路2：动态规划，从时间推移角度看，我如果知道历史最低点，就可以计算出最低点买入和今天卖出的收益。而其中历史最低点无非就是昨天的价格和昨天看到的历史最低点哪个更低。最大收益则是比对今天的收益和历史收益哪个更多。所以买卖需要维护两个量，一个是历史最低点，一个是最大收益。

217:存在重复元素

题干：给你一个整数数组 nums 。如果任一值在数组中出现 至少两次 ，返回 true ；如果数组中每个元素互不相同，返回 false 。

217. 存在重复元素 - 力扣（LeetCode）

思路1：排序法，将数组排序后检查数组左右两个数是否一样，一样就返回True，否则就是False；

思路2：哈希表/字典，将数组送入哈希表，如果插入一个元素时发现该元素已经存在于哈希表中，则说明存在重复的元素；在python中用集合/字典也一样，如果集合长度小于初始长度，则说明存在重复元素。

350：两个数组的交集

题干：给你两个整数数组 nums1 和 nums2 ，请你以数组形式返回两数组的交集。返回结果中每个元素出现的次数，应与元素在两个数组中都出现的次数一致（如果出现次数不一致，则考虑取较小值）。可以不考虑输出结果的顺序。

350. 两个数组的交集 II - 力扣（LeetCode）

思路1：排序后使用双指针，两个指针指向两个序列头，然后比大小。如果一样就添加到队列，如果不一样就挪动一个指针。

思路2：哈希表，在哈希表中存放数字和出现的次数。可以把较短的数组遍历插入，然后再遍历较长的数组，如果碰到表中存在一样的数字，那就将该数字放入结果列表，并将哈希表中的次数擦去一次。

发表于 2023-03-04

前言

​ 在部署完成个人博客后，可以编写文章了，这里推荐使用Typora。虽然说.md文件打开方式有很多，JetBrean系列、vscode甚至记事本都可阅读，但学习记录常常会贴一些图，这个时候用以上软件就不那么方便了，而Typora提供了一个非常方便的解决方案，不仅可以随意插入、预览，它可以连接到图床，使我们设定好图床后一边写就可以一边传了，突出一个丝滑。

​ 在未设置图床之前，我们插入到文档中的图片会保存到本地的一个文件夹。

​ 如果直接将这个md文档作为博客内容发布，别人访问是看不到里面的图片的，而那些下载下来就能看到图片的md文档，里面的图片仔细看都是一个网页连接，这就是我们设置图床的好处。

下载软件

​ 在Typora的偏好设置中，有一个图像设置，其中提供了一个上传服务设定选项，在windows上提供了三个选择，分别是picgo的命令行模式、picgo的app模式、普通命令行模式。如果有硬核的想法，你可以选择用命令行；如果更硬核一点，你也可以自己做一个上传图床的软件，但博主属于一点都不硬核的人，为了操作的简便肯定是选择app模式的。

​ 点击它提供的下载链接，下载它的最新版，点击安装，这个东西装哪里都无所谓。

​ 最后记得将picgo路径设置一下。

​ 我们新建博客可以使用指令hexo new，它会创建一个文章名.md文件到.\blog\source\_posts下，其中他会按照一定的格式创建一个抬头。
